# Decision Framework

이 문서는 HYB Opportunity AI에서 중요한 기능, 설계, 사업 방향을 결정할 때 사용하는 공식 의사결정 기준이다.

---

# 1. 기본 원칙

모든 결정은 다음 순서로 검토한다.

```text
North Star
    ↓
사업 가치
    ↓
사용자 가치
    ↓
기술적 타당성
    ↓
장기 유지보수성
    ↓
현재 우선순위
```

---

# 2. North Star 질문

가장 먼저 다음 질문을 한다.

> 이 결정은 HYB가 더 나은 Opportunity Discovery Platform이 되는 데 기여하는가?

아니라면 우선순위를 다시 검토한다.

---

# 3. 사업 가치

다음 중 하나 이상에 기여하는가?

- 수익 가능성 향상
- Opportunity 발견 능력 향상
- 정확도 향상
- 자동화
- 운영 효율
- 위험 감소
- 확장성

---

# 4. 사용자 가치

사용자는 무엇을 더 쉽게 할 수 있게 되는가?

- 시간을 절약하는가?
- 더 좋은 결정을 내릴 수 있는가?
- 결과를 이해할 수 있는가?
- 신뢰할 수 있는가?

---

# 5. 기술 검토

다음을 확인한다.

- 기존 구조와 충돌하는가?
- 공통 모델을 유지하는가?
- 테스트 가능한가?
- 유지보수 가능한가?
- 더 단순한 방법은 없는가?

---

# 6. 우선순위 판단

작업을 시작하기 전에 확인한다.

- 지금 가장 중요한 문제인가?
- 핵심 기능인가?
- 다음 마일스톤에 도움이 되는가?
- 지금 하지 않으면 어떤 문제가 생기는가?

---

# 7. 구현 전 체크리스트

- [ ] 문제를 명확히 이해했다.
- [ ] 목표가 하나로 정의되어 있다.
- [ ] 영향 범위를 파악했다.
- [ ] 테스트 계획이 있다.
- [ ] 완료 기준이 있다.

---

# 8. 구현 후 체크리스트

- [ ] 정상 동작한다.
- [ ] 기존 기능이 깨지지 않았다.
- [ ] 테스트를 통과했다.
- [ ] 문서를 갱신했다.
- [ ] Git Commit 및 Push 준비가 되었다.

---

# 9. 결정을 미루는 기준

다음과 같은 경우에는 즉시 구현하지 않는다.

- 정보가 부족하다.
- 사업 가치가 불분명하다.
- 장기 구조를 해칠 가능성이 있다.
- 더 중요한 작업이 존재한다.
- 테스트할 수 없다.

---

# 10. 최종 기준

좋은 결정은 단순히 빨리 만드는 결정이 아니다.

좋은 결정은 다음 조건을 만족한다.

- North Star와 일치한다.
- 사용자에게 실제 가치를 준다.
- 장기적으로 유지할 수 있다.
- 설명 가능하다.
- 테스트 가능하다.
- HYB를 더 좋은 제품으로 만든다.
