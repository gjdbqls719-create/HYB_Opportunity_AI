# Development History

## 프로젝트 시작
사용자는 AI를 활용해 온라인에서 저렴한 상품을 찾아 평균가 또는 목표가로 판매하는 자동화 시스템을 만들기로 결정했다.

초기에는 한국 국내 마켓도 검토했지만, 이후 해외에서 상품을 찾아 해외에서 판매하는 구조도 주요 방향으로 받아들였다.

## 개발 환경 구축
- Windows 환경
- VS Code
- Python 가상환경
- Git 설치
- GitHub 저장소
- `.env` 환경변수 관리

여러 컴퓨터에서 프로젝트를 이어가기 위해 Push, Pull, Clone 흐름을 학습했다.

## 초기 구조 설계
프로젝트를 기능별로 분리하기 시작했다.

- 설정
- 외부 마켓 API
- 인증
- 상품 모델
- 상품 매칭
- 수익 계산
- 오케스트레이션
- DB
- 웹 화면

## eBay Sandbox 연결
eBay Sandbox의 애플리케이션 토큰 발급과 Browse API 검색에 성공했다.

`iphone` 검색에서 Sandbox 상품 결과를 가져왔고, 일부 상품을 그룹화하여 기회 분석 결과를 출력했다.

## Opportunity 계산
초기 계산 결과에 다음 항목이 포함되었다.

- 매입 가격
- 예상 판매 가격
- 순이익
- ROI
- 기회 점수
- 검토 판정
- 유사 상품 수

## 문제 해결 경험
개발 과정에서 파일명 오타, import, 들여쓰기, 검색 결과 0건, 폴더 중첩, Git 사용, 화면 스크롤 등 실제 프로젝트 문제를 해결했다.

## 프로젝트 재정비
구조 혼란을 줄이기 위해 새 폴더와 명확한 아키텍처로 정리하는 방향을 선택했다.

README와 문서 체계, 프로젝트 운영 원칙을 정리했다.

## Starter Pack 도입
새 채팅, 새 컴퓨터, 새로운 AI 환경에서도 프로젝트를 이어가기 위해 공식 인수인계 문서인 HYB Starter Pack을 만들기로 결정했다.

## 현재
프로젝트 시작 약 1주일 시점에 기반 구축과 초기 파이프라인 검증을 완료했고, 다음 단계는 최신 코드 Audit과 Opportunity Engine 고도화다.
